## API接続の確認

### ブラウザの開発者ツールで確認

1. **F12**キーで開発者ツールを開く
2. **Network**タブを開く
3. ページをリロード
4. 以下のリクエストが成功しているか確認：
   - `GET http://mameq.xsrv.jp/radish/api/menus` → ステータス200、6件のデータ
   - `GET http://mameq.xsrv.jp/radish/api/orders` → ステータス200、空の配列または注文データ

### Consoleタブで確認

- [ ] エラーメッセージが表示されていない
- [ ] CORSエラーが発生していない
- [ ] API接続エラーが発生していない

## エラーが発生した場合

### メニューが表示されない

1. ブラウザの開発者ツール（F12）を開く
2. **Console**タブでエラーを確認
3. **Network**タブで`/api/menus`へのリクエストを確認
4. APIサーバーが正常に動作しているか確認：
   ```
   http://mameq.xsrv.jp/radish/api/menus
   ```

### 注文が作成できない

1. ブラウザの開発者ツール（F12）を開く
2. **Console**タブでエラーを確認
3. **Network**タブで`/api/orders`へのPOSTリクエストを確認
4. リクエストボディが正しいか確認
5. レスポンスの内容を確認

### CORSエラーが発生する

1. エックスサーバーの`.htaccess`ファイルを確認
2. CORS設定が正しく設定されているか確認
3. `Access-Control-Allow-Origin: *`が設定されているか確認

## テスト手順

### 完全な注文フローのテスト

1. **顧客側で注文**
   - トップページ → お客様用
   - テーブル番号を入力（例: 1）
   - 番号入力で「001」を入力してハンバーガーを追加
   - 番号入力で「003」を入力してコーラを追加
   - カートを確認
   - 注文を確定

2. **店舗側で確認**
   - トップページ → 店舗スタッフ用
   - ログイン（任意のユーザー名・パスワード）
   - 注文管理ページで注文を確認
   - ステータスを「受付済み」に更新
   - ステータスを「調理中」に更新
   - ステータスを「完成」に更新

3. **顧客側で状況確認**
   - 注文状況ページでステータスの変化を確認
   - ステータス進行バーが更新されることを確認

## 完了基準

すべてのチェック項目が✅になれば、システムは正常に動作しています。
